    BACKLOG DO PRODUTO

    Backlog Geral

    O Backlog do Produto é uma lista ordenada de funcionalidades necessárias para o desenvolvimento do sistema. Ele organiza as histórias de usuário (User Stories), que descrevem de forma simples o que o usuário precisa e qual valor a funcionalidade entrega.

    As histórias seguem o formato:
    "Como [usuário], eu quero [funcionalidade] para que [benefício]."

    Priorização do MVP

    Para o MVP (Produto Mínimo Viável), as histórias mais importantes são:

    Acesso e Autenticação

    Essenciais para permitir o uso do sistema e manter segurança básica de acesso.

    US Descrição resumida Prioridade
    US-01 Criar uma conta para acessar o sistema. 🟩 Alta
    US-02 Fazer login com e-mail e senha. 🟩 Alta
    US-03 Recuperar senha esquecida. 🟨 Média
    US-04 Validar código enviado por e-mail. 🟨 Média
    US-05 Redefinir senha. 🟨 Média
    US-06 Exibir mensagens claras de erro e sucesso. 🟩 Alta
    US-07 Navegar entre login, cadastro e recuperação. 🟩 Alta

    MVP 🟩 Incluídos: US-01, US-02, US-06 e US-07

    Postergar 🟨 US-03, US-04 e US-05

    Justificativa:
    Para o MVP, o essencial é permitir cadastro e login. O fluxo completo de recuperação de senha pode ser implementado em versões futuras.

    Sidebar

    Responsável por permitir a navegação entre as principais áreas do sistema, garantindo usabilidade e acesso rápido aos módulos principais.

    US Descrição resumida Prioridade
    US-08 Visualizar uma sidebar fixa com os principais botões de acesso. 🟩 Alta
    US-09 Exibir nome e foto do usuário logado na parte superior da sidebar. 🟨 Média
    US-10 Clicar nos botões da sidebar para acessar as telas correspondentes. 🟩 Alta
    US-11 Destacar o botão da tela atual para indicar a navegação ativa. 🟨 Média
    US-12 Recolher ou expandir a sidebar para economizar espaço em telas menores. 🟥 Baixa
    US-13 Desabilitar ou ocultar itens da sidebar para usuários não autenticados. 🟩 Alta
    US-14 Acessar opções como perfil e logout ao clicar no nome ou foto do usuário. 🟨 Média

    MVP 🟩 Incluídos: US-08, US-10 e US-13

    Postergar 🟨 US-09, US-11 e US-14
    🟥 US-12

    Justificativa:
    Para o MVP, a sidebar precisa existir, permitir navegação entre telas e respeitar o login do usuário.
    Funções adicionais como exibir foto, recolher barra ou acessar opções extras são melhorias visuais e de usabilidade que podem ser adicionadas em versões futuras.
