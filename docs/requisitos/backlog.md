# BACKLOG DO PRODUTO

## Backlog Geral

O Backlog do Produto é uma lista ordenada de funcionalidades necessárias para o desenvolvimento do sistema. Ele organiza as **histórias de usuário** (User Stories), que descrevem de forma simples o que o usuário precisa e qual valor a funcionalidade entrega.  

As histórias seguem o formato:  
**"Como [usuário], eu quero [funcionalidade] para que [benefício]."**

## Priorização do MVP

Para o MVP (Produto Mínimo Viável), as histórias mais importantes são:  

- **US-01 a US-03** (Acesso e autenticação) → garantir que o usuário consiga entrar na plataforma.  
- **US-06 a US-09** (Questões) → permitir que o usuário estude e pratique.
- **US-12** (Estatísticas básicas) → oferecer feedback inicial de desempenho. 
- **US-13** (Evolução ao longo do tempo) → permitir que o usuário veja seu progresso. 
- **US-16** (Administração) → permitir que administradores gerenciem questões.

As demais histórias (US-04, US-05, US-10, US-11, US-14 a US-15) podem ser entregues em incrementos futuros.
